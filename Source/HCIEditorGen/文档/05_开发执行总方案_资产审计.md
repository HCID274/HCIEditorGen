### 10.7 Stage O-SliceO1（Auto-Material Setup：严格契约一键建 MI + 自动挂贴图 + 默认挂载 Slot0）

> 目标：解决“导入后需要手动建材质实例 + 一张张把 BC/N/ORM 贴图连线 + 再手动挂到 Mesh”的高频苦力活。
>
> 第一性原理：这是“内容装配（Content Authoring）写操作”，与“目录/命名整理（Asset Identity & Routing）”属于不同风险域，必须拆为独立 Tool，并在审批卡中展示可审阅的详细清单，且必须 `requires_confirm=true`。

- 本片只做什么（严格契约，先做确定性强的一刀流）：
  - 新增 1 个 Agent Tool：`AutoMaterialSetupByNameContract`（Write，支持 DryRun + Execute）。
  - 契约（Contract）判定（严格）：
    - Mesh：`SM_<ID>`
    - 贴图：`T_<ID>_BC`、`T_<ID>_N`、`T_<ID>_ORM`（允许 `RMA/ARM` 作为 ORM 同义）
  - 输出行为（默认挂载 Slot0，且必须人工确认）：
    - 在目标目录下创建 `Materials/MI_<ID>`
    - Parent 指向夹具 Master：默认读取 `Saved/HCIAbilityKit/TestFixtures/MatLink/latest.json` 的 `snapshot_master_material_object_path`
    - 自动设置 Texture 参数（至少 BC + Normal；ORM 绑定到 MetallicRoughness/Occlusion 等可识别参数）
    - 默认将 Mesh 的 `Material Slot0` 指向新建 `MI_<ID>`
  - 过程可观测性：DryRun/Execute 均提供右下角转圈进度通知（避免“看起来卡死”）。
  - 审批卡展示：必须列出“将创建的 MI 完整路径 + 将修改的 Mesh + 将绑定的参数清单（示例若干）”，不能只显示“归档到某目录”这种摘要。

- 本片不做什么（避免扩散与误杀）：
  - 不做模糊匹配（S2 场景不自动猜测），不做 LLM 二次建议（留给 O2）。
  - 不做多材质槽推断（仅 Slot0），不重建材质图谱，不复制贴图，不做跨目录归并。
  - 不引入 ingest manifest（本片纯目录扫描 + 契约命名驱动）。

### 10.8 Stage O-SliceO2（业务级 Undo：一次审批通过=一条 Undo 记录 + 撤销/重做提示）

> 目标：把“审批通过并执行”的一次业务操作，收敛为 **1 条** UE Undo 记录；美术按一次 `Ctrl+Z` 能整体撤回本次操作。
>
> 第一性原理：大厂管线里，批处理工具如果不能被“像手工操作一样撤销”，美术会不敢用；而“每改一个资产就生成一条 Undo”会让 Undo 历史不可用，因此必须做“业务级事务”。

- 本片只做什么：
  - Commit 执行（`DryRun=false` 且 `bUserConfirmedWriteSteps=true`）时，用 `FScopedTransaction` 包裹整次 Plan 执行：
    - `TransactionContext="HCIAbilityKit"`
    - `SessionName="HCIAbilityKit: <intent> (<request_id>)"`
  - 注册 `FEditorUndoClient`，仅匹配 `TransactionContext=="HCIAbilityKit"` 的事务：
    - `PostUndo/PostRedo` 右下角 Toast 提示（非阻塞、非 modal）。
  - UI 假死缓解：在批处理循环中周期性泵消息（`PumpMessages`），保证执行期间窗口不进入“未响应”。

- 本片不做什么：
  - 不新增“自动全量回退”（不替代版本控制 Revert）。
  - 不实现“撤销上一次 HCI 操作”单独按钮（仅依赖 UE 原生 Undo/Redo 快捷键与菜单）。

- UE 手测门禁（O2）：
  1. 运行：`HCIAbilityKit.MatLinkReset`
  2. 打开：`HCIAbilityKit.AgentChatUI`，对 `S1_PerfectContract` 执行一次真实写操作（如 `AutoMaterialSetupByNameContract`）。
  3. 打开菜单：`Edit -> Undo History`：
     - 预期：新增 1 条记录，标题形如 `HCIAbilityKit: auto_material_setup_by_contract (req_...)`（intent/request_id 允许不同，但必须是 1 条）。
  4. 按一次 `Ctrl+Z`：
     - 预期：触发撤销成功（材质实例/挂载恢复到执行前状态），并出现 Toast：`已撤销：HCIAbilityKit ...`。
  5. 按一次 `Ctrl+Y`（或菜单 Redo）：
      - 预期：触发重做成功，并出现 Toast：`已重做：HCIAbilityKit ...`。

### 10.9 Stage O-SliceO3（孤儿资产可定位：Auto-Material 失败/退出时输出 Orphans 并一键定位）

> 目标：当严格契约无法建立唯一关联（例如 S2_Fuzzy / S3_None）时，系统必须把“孤儿资产/未解决资产”显式列出来，并允许美术**点击定位到 Content Browser**，快速找到“命名/目录哪里乱了”。
>
> 第一性原理：真实项目里，自动化最常见的阻塞点不是“不会执行”，而是“失败后美术不知道该改哪里”。因此失败路径必须提供**可定位的证据链**，而不是一句“失败”。

- 本片只做什么：
  - `AutoMaterialSetupByNameContract` 的 DryRun/Execute 无论成功与否，都输出两类证据字段（Evidence）：
    - `orphan_assets`：未命中契约解析的资产（例如贴图后缀不符合 `BC/N/ORM`）。
    - `unresolved_assets`：命中部分线索但无法配齐一组的资产（例如 `SM_Old_Barrel` 找到 Mesh 但缺贴图）。
  - 将上述证据字段接入现有 LocateTargets 机制：
    - AgentChatUI 在执行后弹出可点击的定位按钮（LocateTargets 气泡）。
    - 点击按钮可在 Content Browser 中定位对应资产（`SyncBrowserToObjects`）。

- 本片不做什么：
  - 不做 LLM 模糊匹配建议（仅把问题暴露出来并让美术快速定位）。
  - 不做“孤儿资产专属选项卡面板”（先复用 LocateTargets 气泡；如需全量列表再另切片做 UI）。

- UE 手测门禁（O3）：
  1. 运行：`HCIAbilityKit.MatLinkReset`
  2. 打开：`HCIAbilityKit.AgentChatUI`，对 `S2_Fuzzy` 输入：
     - `扫描 /Game/__HCI_Test/Incoming/MatLinkChaos/S2_Fuzzy，然后按契约自动创建材质实例并挂贴图（仅预览）`
  3. 预期（失败但可定位）：
     - DryRun 明确失败或提示“契约不足/缺贴图”（允许失败，但不得写入资产）。
     - 聊天窗口出现 LocateTargets 气泡，包含 `orphan_assets/unresolved_assets` 的定位按钮（数量小于等于 12 即可）。
     - 点击任意按钮：Content Browser 自动定位到该资产。

### 10.10 Stage O-SliceO4（孤儿原因可解释：LocateTargets 二行展示“为什么不通过”）

> 第一性原理：失败不可怕，可怕的是失败后无法行动。失败路径必须把“可行动的原因”写清楚（例如：后缀不满足契约 / 缺少配对 Mesh / 缺少 BC/N/ORM）。
