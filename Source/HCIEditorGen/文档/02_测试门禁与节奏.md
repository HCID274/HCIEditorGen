# HCIAbilityKit 测试门禁与节奏

> 状态：冻结（M0-V5）
> 更新时间：2026-02-24

## 1. 总体规则（强制）

- 每完成一个切片必须停下，等待用户 UE 手测。
- 用户未明确 `Pass`，不得进入下一切片。
- 每次验证必须有测试记录文件。
- 写操作必须覆盖失败路径验证。

## 2. 标准切片流程

1. 冻结切片目标与非目标。
2. 实现并本地编译通过。
3. 输出 UE 手测命令与 Pass/Fail 判据。
4. 用户执行 UE 手测并反馈。
5. 回写测试记录后再推进。

## 3. 当前阶段门禁

### G_GATE（已通过）

- `G10` 已通过：执行就绪成功路径 + 阻断路径（`E4218/E4219/E4005/E4202`）均已验证。

### H1_GATE（当前必须执行）

#### H1_GATE_01 LLM 成功出计划

- 步骤：输入命中三类意图（命名溯源、关卡排雷、资产规范）。
- 预期：
  - 输出合法 Plan JSON；
  - `planner_provider=llm`；
  - `plan_validation=ok`。

#### H1_GATE_02 LLM 超时回退

- 步骤：模拟 LLM 超时。
- 预期：
  - 自动回退关键词规划器；
  - `planner_provider=keyword_fallback`；
  - `fallback_reason=llm_timeout`；
  - 整体不崩溃。

#### H1_GATE_03 LLM 非法结构回退

- 步骤：模拟 LLM 返回非法 JSON 或缺失必填字段。
- 预期：
  - 回退关键词规划器；
  - 命中 `E4302` 或 `E4001`（以最终实现口径为准）；
  - 日志可定位 `field/reason/hint`。

#### H1_GATE_04 白名单与门禁不被绕过

- 步骤：构造 LLM 输出非白名单 tool 或风险字段不一致。
- 预期：
  - 由 `PlanValidator/ExecutionGate` 拦截；
  - 分别命中 `E4002`/`E4003`；
  - 无真实写入。
