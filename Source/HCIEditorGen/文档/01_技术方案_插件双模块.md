# HCIAbilityKit 技术方案（插件双模块，冻结版）

> 状态：冻结（V2）  
> 更新时间：2026-02-24  
> 适用范围：`HCIAbilityKit` 单模块

## 1. 目标与边界

- 目标：在 UE 编辑器内交付“资产审计主链路 + Agent 安全编排链路（以 dry-run 为主）”。
- 本期聚焦：资产规范执行、性能风险发现、数据校验与可追踪审阅。
- 非目标：无确认自动批量改写、审批系统 UI 平台化、多项目治理平台。

## 2. 模块架构（冻结）

- `HCIAbilityKitRuntime`
  - 职责：数据契约、解析服务、扫描与规则、报告序列化、Plan/执行契约与校验。
  - 约束：不得依赖 Editor-only 模块。
- `HCIAbilityKitEditor`
  - 职责：Factory/Reimport、控制台命令、菜单与交互、UE 定位与演示入口。
  - 依赖：只允许 `Editor -> Runtime` 单向依赖。

## 3. 主链路数据流

1. Import/Reimport：`Factory -> ParserService -> Asset`。
2. AuditScan：`AssetRegistry/FAssetData -> AuditScanService -> RuleRegistry -> AuditReport`。
3. Export：`AuditReport -> JsonSerializer`。
4. Agent Plan：`NL -> Planner -> Plan JSON -> PlanValidator`。
5. Agent Execute：`Executor(simulate_dry_run) -> Preflight Gates -> StepResults`。
6. Review：`RunResult -> DryRunDiffReport -> Locate(camera_focus/sync_browser)`。
7. 申请链路：`Review -> ApplyRequest -> ConfirmRequest -> ExecuteTicket -> Sim* 收敛包`。
8. Stage G（当前）：继续维持 dry-run 收敛链路，`G10` 文档先行冻结后再实现。

## 4. 核心契约（冻结）

- Tool Registry：一期白名单 7 工具  
  `SetTextureMaxSize / SetMeshLODGroup / ScanLevelMeshRisks / NormalizeAssetNamingByMetadata / RenameAsset / MoveAsset / ScanAssets`
- 执行门禁：
  - Confirm Gate（未确认写入阻断）
  - Blast Radius（`MAX_ASSET_MODIFY_LIMIT=50`）
  - All-or-Nothing 事务策略
  - SourceControl Fail-Fast（含离线本地模式）
  - Mock RBAC（未命中用户默认 `Guest(read_only)`）
  - LOD Safety（非 StaticMesh 或 Nanite 启用阻断）
- 定位策略：
  - Actor：`Camera Focus`
  - 纯资产：`SyncBrowserToObjects`

## 5. 审计能力分层

- Stage C 已落地规则（Runtime RuleRegistry）：
  - `TriangleExpectedMismatchRule`
  - `TextureNPOTRule`
  - `HighPolyAutoLODRule`
- 关卡排雷与命名溯源能力：
  - 通过 Tool/Plan/Executor 链路承载（`ScanLevelMeshRisks`、`NormalizeAssetNamingByMetadata` 等）。

## 6. 质量红线

- Runtime 禁止依赖 Editor-only。
- 失败路径禁止脏写、半写。
- 错误必须可定位（文件/字段/原因/建议）。
- 未经验证不得标记“完成”。
