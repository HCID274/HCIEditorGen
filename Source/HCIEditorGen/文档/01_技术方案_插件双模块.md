# HCIAbilityKit 技术方案（插件双模块，冻结版）

> 状态：冻结（V3）  
> 更新时间：2026-02-24  
> 适用范围：`HCIAbilityKit`

## 1. 目标与边界

- 目标：交付 `资产审计主链路 + Agent 安全编排`，以可审计 dry-run 为核心。
- 当前阶段：`Stage H`（H1 已通过，H2 进行中：稳定性增强）。
- 非目标：审批流平台化 UI、未经确认自动批量改写。

## 2. 模块架构（冻结）

- `HCIAbilityKitRuntime`
  - 数据契约、解析/校验、扫描规则、Plan/Execute 契约。
  - 禁止依赖 Editor-only 模块。
- `HCIAbilityKitEditor`
  - Factory/Reimport、命令入口、定位交互、UE 集成。
- 依赖方向固定：`Editor -> Runtime`。

## 3. 主链路数据流（当前）

1. `Import/Reimport`: `Factory -> ParserService -> Asset`
2. `AuditScan`: `AssetRegistry -> RuleRegistry -> AuditReport`
3. `Agent Plan`: `NL -> Planner(Keyword/LLM) -> Plan JSON -> PlanValidator`
4. `Agent Execute`: `Executor(simulate_dry_run) -> Preflight Gates -> StepResults`
5. `Review`: `RunResult -> DryRunDiffReport -> Locate`
6. `Stage G`: 执行链路收口（G1~G10 已完成）
7. `Stage H`: Planner 增强（H1 接入完成，H2 增加 retry/circuit/metrics）

## 4. 一期冻结能力

- Tool 白名单（7）：
  - `SetTextureMaxSize`
  - `SetMeshLODGroup`
  - `ScanLevelMeshRisks`
  - `NormalizeAssetNamingByMetadata`
  - `RenameAsset`
  - `MoveAsset`
  - `ScanAssets`
- 门禁链：`Confirm`、`BlastRadius(50)`、`All-or-Nothing`、`SC Fail-Fast`、`Mock RBAC`、`LOD Safety`。
- 定位策略：Actor=`Camera Focus`，资产=`SyncBrowserToObjects`。

## 5. H1~H2 架构约束（新增冻结）

- 只新增 Planner Provider 层，不改执行链。
- Provider 优先级：`LLM -> KeywordFallback`。
- LLM 返回非法时必须回退并记录原因。
- H2 增加稳定性控制：`retry + circuit breaker + metrics`，仍不改变执行链语义。
- 不允许 LLM 直接触发执行，仅生成候选 Plan。
