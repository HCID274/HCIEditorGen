## 3. 最近完成切片：Stage H-SliceH2（已通过 UE 手测）

### 3.1 本片只做什么

- 补齐 LLM 稳定性策略：`timeout -> retry -> circuit breaker -> metrics`。
- 保留现有关键词规划器作为回退路径，不改变 F 链/G 链语义。
- 扩展 Planner 结果元数据（尝试次数/重试/熔断/连续失败计数）。

### 3.2 本片不做什么

- 不修改 ToolRegistry 白名单。
- 不修改 ExecutionGate 语义。
- 不引入真实写执行，不做审批 UI。
- 不接入真实线上 LLM API（仍使用本地 mock seam）。

### 3.3 实现边界（代码级）

- Runtime：`FHCIAbilityKitAgentPlannerBuildOptions` 新增 `LlmRetryCount/CircuitBreaker*/LlmMockFailAttempts`。
- Runtime：新增熔断状态与指标快照接口（`GetMetricsSnapshot/ResetMetricsForTesting`）。
- Runtime：新增 `llm_circuit_open -> E4305` 回退口径。
- Editor：新增 H2 演示命令与指标命令。
- 全链路仍需经过：`PlanValidator -> Executor(simulate_dry_run)`。

### 3.4 H2 验收日志口径（唯一判据）

- 成功：
  - `planner_provider=llm`
  - `llm_attempts>=1`
  - `plan_validation=ok`
- 回退：
  - `planner_provider=keyword_fallback`
  - `fallback_reason in {llm_timeout,llm_invalid_json,llm_contract_invalid,llm_empty_response,llm_circuit_open}`
  - 熔断场景命中 `E4305`
- 指标：
  - 可观测字段完整：`total_requests/llm_preferred_requests/llm_success_requests/keyword_fallback_requests/retry_used_requests/retry_attempts/circuit_open_fallback_requests/consecutive_llm_failures`
- 安全：
  - 非白名单/风险不合法仍返回 `E4002/E4003`
  - 不发生真实写入
