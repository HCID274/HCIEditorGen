## 3. 历史参考切片：Stage H-SliceH3（已关闭）

### 3.1 本片只做什么

- 真实接入 LLM Provider：`C++ + FHttpModule`，请求 OpenAI Compatible Chat Completions。
- 认证改为本地配置文件读取：`Saved/HCIAbilityKit/Config/llm_provider.local.json`。
- 保留现有关键词规划器 fallback 与 H2 的 `retry/circuit/metrics` 机制。
- 全链路继续 `PlanValidator -> Executor(simulate_dry_run)`。

### 3.2 本片不做什么

- 不修改 ToolRegistry 白名单。
- 不修改 ExecutionGate 语义。
- 不引入真实写执行，不做审批 UI。
- 不引入 Python 主调用链，不走外部脚本桥接。

### 3.3 实现边界（代码级）

- Runtime：新增真实 HTTP Provider（DashScope OpenAI compatible endpoint）。
- Runtime：新增 LLM 响应解析（`choices[0].message.content -> Plan JSON`），并落到现有 `FHCIAbilityKitAgentPlan`。
- Runtime：新增配置缺失与 HTTP 失败收敛口径（错误码 + fallback）。
- Editor：新增真实 LLM 演示命令（用于 UE 手测）。

### 3.4 H3 验收日志口径（唯一判据）

- 成功：
  - `planner_provider=llm`
  - `provider_mode=real_http`
  - `plan_validation=ok`
- 回退（配置）：
  - `planner_provider=keyword_fallback`
  - `fallback_reason=llm_config_missing`
- 回退（运行时）：
  - `planner_provider=keyword_fallback`
  - `fallback_reason in {llm_timeout,llm_invalid_json,llm_contract_invalid,llm_circuit_open}`
- 回退（传输）：
  - `planner_provider=keyword_fallback`
  - `fallback_reason=llm_http_error`
  - `plan_validation=ok`
- 指标：
  - 可观测字段完整：`total_requests/llm_preferred_requests/llm_success_requests/keyword_fallback_requests/retry_used_requests/retry_attempts/circuit_open_fallback_requests/consecutive_llm_failures`
- 安全：
  - 非白名单/风险不合法仍返回 `E4002/E4003`
  - 不发生真实写入

## 4. 结构解耦与技术债清理（No Semantic Changes）

### 4.1 Phase1（已完成）

- `Task1`：同步 `real_http` 伪支持退役，强制真实 LLM 请求走异步链路（同步路径返回 `E4310`）。
- `Task2`：Planner 职责解耦（`PromptBuilder + LLMClient`）；Prompt Bundle 读取与配置加载去重。
- `Task3`：Commands 拆分为分域注册文件，状态迁移至集中状态容器，减少静态全局耦合。
- `Task4`：ToolRegistry 读取路径收敛到只读访问语义，降低跨命令重置污染风险。
- 门禁：UE 手测 `Pass` + 全量编译通过（已达成）。
