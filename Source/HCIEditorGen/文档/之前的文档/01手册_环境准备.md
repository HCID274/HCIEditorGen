# UE实践手册：环境准备（UE5.4.4 / MSVC / Rider）

目标：把“环境问题”一次性清零。你只要严格照做，后续工具链与 Runtime dogfooding 就不会被安装/编译/IDE 折磨。

> 本手册是通用前置条件：`HCIEditorGen`（工具链）与 `HCIDuel`（Lyra 底座）都依赖它。

---

## 1. Epic Games Launcher（安装与设置）

1. 安装 Epic Games Launcher（官网下载安装包，安装完成后登录）。
2. 进入 Launcher 后，确认左侧有 `Unreal Engine` 入口（不是只停留在 Store）。
3. 建议把引擎安装到 SSD，路径尽量短（示例：`D:\UE\UE_5.4`）。

---

## 2. 安装 Unreal Engine 5.4.4（重点：一定要勾选调试符号）

1. Epic Launcher → `Unreal Engine` → `Library`。
2. 在 `Engine Versions` 区域点击 `+` 新增一个版本槽位，选择 `5.4.4`（或 5.4.x 的最新补丁版，但推荐固定到 5.4.4）。
3. 点击该版本右侧的 `Options...`，勾选：

   * `Editor symbols for debugging`（必须）
   * `Starter Content`（可选；新手建议勾上，省去找测试资源的时间）
   * 其他项先不强求（MVP 期以稳定为先）
4. 点击 `Install`，等待完成。

空间预估（保守）：

* 引擎本体 + 调试符号：通常需要几十 GB 以上（取决于你勾选项），建议预留 120GB 以上可用空间，避免中途爆盘。

---

## 3. 安装 C++ 工具链（即使用 Rider 也必须装）

> Rider 负责写代码，但底层编译器/SDK 来自 MSVC。

推荐路径（任选其一即可）：

1. Visual Studio 2022（完整版）安装 `Desktop development with C++` 工作负载；或
2. Visual Studio 2022 Build Tools（轻量）安装同等 C++ 组件。

必备组件（关键字）：

* MSVC v143 工具集（C++ 编译器）
* Windows 10/11 SDK（任意一个完整 SDK 版本即可）

验证方式：

* 以后在 Rider 里 Build `Development Editor` 能通过，就说明工具链齐全。

---

## 4. 安装 Rider（建议用 Toolbox 管理）

1. 安装 JetBrains Toolbox → 安装 Rider（保持更新渠道稳定）。
2. Rider 首次打开 UE 项目时，通常会提示安装/启用 RiderLink（把 UE 与 Rider 的蓝图引用/跳转/调试链路打通）。

   * 建议选择“安装到 Project”（而不是 Engine），避免污染引擎安装目录，也方便项目迁移。
   * 安装完成后按提示重启 Rider/UE Editor。

---

## 5. 安装验证（必须做完这 3 个验证再开工）

1. Epic Launcher 里能看到 `UE 5.4.4` 已安装，并且可以 `Launch`。
2. UE Editor 能正常启动到主界面（不崩溃）。
3. 新建一个 **C++** Third Person 项目后：

   * Rider 打开 `.uproject` 能完成索引
   * Build 一次 `Development Editor` 通过
   * 回到 UE 点击 `Play (PIE)` 角色能移动

常见安装坑（先记住，遇到直接按这里排查）：

* Build 报 MSVC/SDK 缺失：就是第 3 节没装齐，回去补安装。
* UE/项目路径太深：把引擎/工程移动到短路径（`D:\UE\...` / `D:\Projects\...`）。
* UE 宏红线（IntelliSense 报错）：以“能编译通过”为准，别被假红线影响心态。
